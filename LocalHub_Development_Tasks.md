# LocalHub 개발 작업 정리

## 프로젝트 개요
신뢰할 수 있는 진짜 지역 정보의 부재와 높은 탐색 피로도(광고성 정보 범람 및 후기 확인의 어려움)를 감소시키기 위해 부산을 방문하는 관광객 및 지역 주민을 대상으로 하는 웹페이지를 개발하는 프로젝트입니다. WBS 일정과 개발 순서를 반영하여 기획/초기세팅(Day 1), 핵심 기능 구현(Day 2), 최종 연동 및 배포(Day 3) 단계를 유기적으로 설계하였습니다.

---

# 프론트엔드 (Frontend)

## 1. 프로젝트 초기 세팅 (Day 1)
프로젝트 개발을 위한 기본 환경을 구성하고 전체 페이지의 공통 구조를 설계합니다.

### 해야 할 일
- [ ] **Vue3 프로젝트 생성 및 폴더 구조 구성**
  - Vite를 사용하여 빠르고 가벼운 Vue3 개발 환경을 시작하고 컴포넌트, 뷰, 스토어 등의 기본 폴더 구조를 표준화합니다.
- [ ] **Vue Router 및 Pinia 상태 관리 설정**
  - 페이지 이동 경로(라우팅)를 정립하고 전역 상태(사용자, 챗봇 대화 기록 등) 관리를 위한 저장소를 구성합니다.
- [ ] **공통 레이아웃(Header, Footer, Navigation) 구현**
  - 서비스의 일관된 UI/UX를 제공하기 위해 전체 페이지에서 공통으로 보일 상하단 바 및 내비게이션 바를 구현합니다.
- [ ] **API 통신(Axios) 및 환경변수(.env) 설정**
  - 백엔드 FastAPI 서버와의 통신을 관리할 공통 Axios 인스턴스를 정의하고 개발/실서버 환경 주소를 분리합니다.

---

## 2. 게시판 화면 구현 및 API 연동 (Day 2)
익명 게시글의 조회, 작성, 수정, 삭제가 가능한 사용자 인터페이스를 구축하고 백엔드와 실시간으로 데이터를 연동합니다.

### 해야 할 일
- [ ] **게시글 목록 및 상세 페이지 UI 구현**
  - 게시글 번호, 제목, 조회수, 작성일 등이 포함된 목록 화면을 구현하고 각 글을 선택했을 때 나타나는 상세 보기 페이지를 레이아웃합니다.
- [ ] **게시글 작성 및 수정 폼 구현**
  - 익명 사용자가 자유롭게 의견을 남길 수 있도록 제목, 내용, 그리고 향후 본인 확인을 위한 평문 비밀번호 입력 창을 설계합니다.
- [ ] **비밀번호 검증 모달 및 삭제 기능 연동**
  - 게시글의 수정 및 삭제 버튼 클릭 시, 작성 시 입력했던 비밀번호를 다시 입력받는 검증 팝업 컴포넌트를 구현합니다.
- [ ] **게시판 CRUD API 실데이터 연동**
  - Axios를 호출하여 게시글 작성(POST), 목록 조회(GET), 수정(PUT), 삭제(DELETE)가 서버 및 DB와 정상적으로 연동되도록 바인딩합니다.

---

## 3. 게시판 검색·조회수 및 지도 시각화 (Day 2)
게시판의 편의 기능을 개선하고, 관광지 위치 정보를 활용해 직관적인 지도 탐색 화면을 구축합니다.

### 해야 할 일
- [ ] **키워드 검색 및 페이지네이션 UI 구현**
  - 사용자가 제목 및 내용을 기반으로 글을 검색할 수 있도록 필터를 적용하고, 페이지 단위로 목록이 끊겨 보이도록 페이징 컴포넌트를 부착합니다.
- [ ] **Kakao Maps API 연동 및 설정**
  - 카카오 맵 외부 자바스크립트 라이브러리를 로드하고 프론트엔드 내 지도가 렌더링될 영역과 기본 설정을 마칩니다.
- [ ] **관광지 좌표 데이터 기반 지도 마커 시각화**
  - 백엔드에서 내려주는 관광지 리스트 정보의 위도(mapy), 경도(mapx) 좌표를 활용해 지도 위에 위치 핀(마커)을 꽂아 보여줍니다.

## 4. 챗봇 UI 및 관광지 연동 기능 (Day 2~3)
OpenAI 추천 시스템과 실시간으로 대화할 수 있는 전용 UI와 추천 장소 바로가기 컴포넌트를 개발합니다.

### 해야 할 일
- [ ] **실시간 대화형 채팅 화면 UI 개발**
  - 사용자 질문과 챗봇의 응답이 말풍선 형태로 자연스럽게 이어지도록 채팅 위젯 및 스크롤 최적화를 구현합니다.
- [ ] **LocalStorage 기반 대화 기록 관리**
  - 서버 DB 세션에 저장하지 않는 챗봇의 기획 특성을 고려해, 브라우저가 꺼져도 이전 대화 기록이 유지되도록 로컬 저장소와 결합합니다.
- [ ] **추천 관광지(related_places) 카드 UI 연동**
  - 챗봇이 추천해 준 답변 속에 포함된 실시간 매칭 관광지 데이터를 카드 형태의 미니 컴포넌트로 동적 렌더링합니다.

## 5. 최종 통합 및 배포 준비 (Day 3)
개별적으로 완성된 컴포넌트들을 합치고 빌드 과정을 거쳐 상용 호스팅 환경에 반영합니다.

### 해야 할 일
- [ ] **전체 페이지 라우팅 최종 검증**

  - 관광지에 직접 방문 중인 사용자가 모바일로도 원활히 볼 수 있도록 화면 해상도에 맞춘 최적화 레이아웃을 마감합니다.

# 백엔드 (Backend)

## 1. 프로젝트 초기 세팅 및 DB 구축 (Day 1)
FastAPI의 개발 토대를 다지고, 데이터를 영구 저장할 SQLite 데이터베이스 모델 스키마를 구성합니다.

### 해야 할 일
- [ ] **FastAPI 기본 프로젝트 아키텍처 설계**
  - 가볍고 직관적인 FastAPI의 장점을 살려 라우터별로 파일(posts, places, chatbot)을 구조화하여 확장성을 확보합니다.
- [ ] **CORS 미들웨어 적용 및 공통 설정**
  - 프론트엔드 도메인(Netlify 등)에서 오는 API 호출이 보안 오류 없이 승인되도록 CORS 설정을 구성합니다.
- [ ] **places (관광지) 테이블 스키마 정의 및 DB 생성**
  - contentid(PK), title, mapx, mapy 등 25개의 세부 정보를 담을 수 있는 SQLite 테이블 스키마를 생성하고 이식합니다.
- [ ] **posts (익명 게시판) 테이블 스키마 정의 및 DB 생성**
  - post_id(PK), title, content, password(평문), view_count(기본값 0) 및 생성/수정일시 필드를 구축합니다.

---

## 2. 백엔드 기본 핵심 아키텍처 설계 (Day 1~2)
전체 시스템의 비즈니스 로직 신뢰성을 높이고 에러에 안정적으로 대응하는 프레임워크 설정을 진행합니다.

### 해야 할 일
- [ ] **환경변수(.env) 보안 및 API 문서화(Swagger) 점검**
  - OpenAI API Key나 DB 파일 경로와 같은 중요 정보가 노출되지 않도록 설정하고, 기본 제공되는 Swagger UI로 엔드포인트를 통제합니다.
- [ ] **글로벌 예외 처리기(Exception Handler) 설계**
  - DB 조회 실패, 존재하지 않는 리소스 요청 등 다양한 예외 상황 시 클라이언트가 해석할 수 있는 통일된 JSON 에러 메시지 규격을 정의합니다.

---

## 3. 게시판 CRUD 핵심 API 구현 (Day 2)
익명 게시판이 원활하게 작동할 수 있도록 핵심 데이터 조작 기능을 설계하고 비즈니스 규칙을 구현합니다.

### 해야 할 일
- [ ] **게시글 목록 조회 API (`GET /api/posts`)**
  - 최신 작성일(`created_at` 역순) 순으로 정렬된 데이터를 전송하고, 매개변수를 통해 페이지네이션 기능을 제공합니다.
- [ ] **게시글 상세 조회 API (`GET /api/posts/{id}`)**
  - 해당 ID의 글 내용을 전송하는 동시에, 데이터베이스 상의 `view_count`(조회수) 필드를 즉시 1 증가시키는 업데이트 구문을 처리합니다.
- [ ] **신규 글 생성 API (`POST /api/posts`)**
  - 제목, 내용, 평문 비밀번호를 받아 새 게시글 레코드를 데이터베이스에 삽입합니다.
- [ ] **비밀번호 검증 기반 수정/삭제 API (`PUT/DELETE /api/posts/{id}`)**
  - 프론트엔드에서 비밀번호를 전송받아 DB 내의 평문 비밀번호와 직접 일치 여부를 검사한 후, 성공 시에만 수정 또는 삭제 처리를 승인합니다.

---

## 4. 관광지 및 챗봇 연동 API 구현 (Day 2~3)
places 데이터베이스를 탐색하고, OpenAI 인공지능을 연동하여 실시간 부산 맞춤 정보 추천 시스템을 완성합니다.

### 해야 할 일
- [ ] **관광지 검색 및 좌표 제공 API**
  - 지역 구분, 분류 코드, 키워드를 인자로 받아 해당하는 places 목록을 가공해 좌표(위/경도)와 함께 리턴합니다.
- [ ] **OpenAI SDK 연결 및 챗봇 연드포인트 구현 (`POST /api/chatbot/query`)**
  - 환경변수의 시크릿 키를 이용해 OpenAI API 클라이언트를 세팅하고 사용자 메시지를 전달받는 엔드포인트를 구현합니다.
- [ ] **프롬프트 엔지니어링 및 관련 관광지(related_places) 매칭 가공**
  - AI가 부산의 최신 맞춤 정보를 자연스럽게 답하게 유도하고, 답변 중 등장한 명소를 DB에서 실시간 검색하여 상세 정보 데이터를 답변 포맷에 동적으로 주입 및 결합합니다.

## 5. 최종 검증 및 배포 (Day 3)
백엔드 기능의 결함을 검사하고, 클라우드 환경에 호스팅하여 프론트엔드와 최종 통신을 성립시킵니다.

- [ ] **Render 플랫폼 배포 및 서버 상용화**
