# LocalHub 페이지별 기능명세서

## 1. 문서 정보

| 항목 | 내용 |
| --- | --- |
| 문서 버전 | 1.0 |
| 작성일 | 2026-07-15 |
| 작성 기준 | 현재 소스 코드 기준(As-Is) |
| 작성 범위 | 라우터에 등록된 7개 페이지, 공통 레이아웃, 공통 챗봇 위젯 |
| 제외 범위 | 라우터에 연결되지 않은 `ChatbotView.vue`, `PlacesView.vue` |
| 목적 | 화면 개발, API 연동, QA 테스트에서 공통으로 사용할 기능 기준 정의 |

### 1.1 표기 기준

| 표기 | 의미 |
| --- | --- |
| 현재 | 소스 코드에서 동작이 확인된 기능 |
| 보완 필요 | 기능은 있으나 예외 처리나 사용성이 부족한 상태 |
| 미구현 | 화면 또는 로직이 현재 구현되어 있지 않은 상태 |

## 2. 페이지 공통 작성 항목

모든 페이지는 다음 항목을 기준으로 명세한다.

| 공통 항목 | 작성 내용 |
| --- | --- |
| 페이지 ID | 페이지를 구분하는 고유 식별자 |
| 페이지명 | 사용자에게 표시되는 화면의 명칭 |
| 접근 URL | 프론트엔드 라우터 기준 URL |
| 목적 | 해당 화면에서 사용자가 달성하는 주요 목표 |
| 선행 조건 | 페이지 접근 및 기능 수행 전에 필요한 조건 |
| 데이터/API | 화면이 사용하는 백엔드 API 또는 정적 데이터 |
| 입력폼 | 입력 전, 입력 중, 유효/무효 상태와 검증 규칙 |
| 버튼 | 비활성화, 활성화, 선택 상태 및 클릭 결과 |
| 로직 | 데이터 정렬, 필터링, 변환 및 화면 표시 규칙 |
| 성공 결과 | 정상 처리 후 메시지, 화면 갱신 또는 이동 결과 |
| 빈 상태 | 표시할 데이터가 없을 때의 화면 |
| 로딩 상태 | 요청 처리 중 사용자에게 제공하는 상태 |
| 에러 상황 | 잘못된 입력, API 실패, 외부 자원 실패 처리 |
| 제한 개수 | 조회·표시·입력 가능한 데이터 수 또는 길이 |
| 외부 액션 | 외부 사이트, 지도 타일, 브라우저 기능과의 연동 |
| 반응형 | 모바일·태블릿·데스크톱에서의 배치 규칙 |
| 테스트 조건 | 기능 완료 여부를 판단하는 검수 기준 |

## 3. 시스템 공통 규칙

| 항목 | 현재 규칙 | 상태 및 비고 |
| --- | --- | --- |
| 프론트 기본 API 주소 | `http://localhost:8000` | 환경 변수 설정이 없을 때 사용 |
| 요청/응답 형식 | JSON | 게시글 삭제 요청도 JSON body 사용 |

| 데이터베이스 | SQLite `backend/localhub.db` | 현재 게시글과 장소 데이터 저장 |

| CORS | 로컬 개발 Origin 허용 | 배포 환경 Origin 추가 필요 |
| 공통 에러 화면 | 없음 | 라우터 404 페이지와 전역 오류 UI 미구현 |
| 공통 로딩 UI | 없음 | 페이지별 스피너·스켈레톤 미구현 |
| 날짜 표시 | 화면별 로컬 형식 변환 | 시간대와 형식의 통일 규칙 보완 필요 |

### 5.2 기능 명세

| 기능 ID | 항목 | 설명 | 상태 |

| HOME-02 | 주요 기능 이동 | 게시판·지도 기능 버튼 클릭 시 해당 페이지로 이동한다. | 현재 |

| HOME-07 | 전체 지도 이동 | 지도 영역의 이동 버튼을 누르면 `/map`으로 이동한다. | 현재 |

### 5.3 상태 및 예외 처리

| 상태 | 처리 내용 | 판정 |

- 최근 게시글은 최대 4개, 지도 마커는 정적 데이터 11개, 카테고리 카드는 3개를 표시한다.
- 홈 지도는 전체 장소 데이터나 장소 API가 아닌 `mockData.js`를 사용한다.
- 카테고리 카드 영역에는 별도의 모바일 전용 열 재배치 규칙이 없어 작은 화면에서 확인이 필요하다.

## 6. PAGE-02 게시판 목록

### 6.1 기본 정보

| 항목 | 내용 |
| --- | --- |
| 페이지 ID | PAGE-02 |
| 페이지명 | 게시판 목록 |
| 접근 URL | `/board` |
| 목적 | 게시글을 검색·정렬·탐색하고 글쓰기 또는 상세 화면으로 이동한다. |
| 선행 조건 | 없음 |
| 데이터/API | `GET /api/posts?page=1&page_size=100` |

### 6.2 입력폼 및 버튼

| 기능 ID | 항목 | 설명 | 상태 |
| --- | --- | --- | --- |
| BOARD-01 | 검색 입력 | 제목과 내용을 대상으로 대소문자 구분 없이 즉시 검색한다. | 현재 |
| BOARD-02 | 정렬 선택 | 최신순 또는 조회순을 선택한다. | 현재 |
| BOARD-03 | 글쓰기 버튼 | `/write`로 이동한다. | 현재 |
| BOARD-04 | 게시글 행 | 클릭한 게시글의 `/board/{post_id}`로 이동한다. | 현재 |
| BOARD-05 | 페이지 버튼 | 현재 필터 결과를 페이지당 4개씩 나누어 표시한다. | 현재 |

### 6.3 로직

- 최초 요청으로 게시글을 최대 100개 가져온 뒤 검색, 정렬, 페이지 구분을 프론트에서 처리한다.
- 최신순은 작성일이 아니라 `post_id` 내림차순으로 정렬한다.
- 조회순은 `view_count` 내림차순으로 정렬한다.
- 검색어는 제목 또는 내용 중 하나라도 포함하면 결과에 포함한다.
- 검색이나 정렬 변경 시 현재 페이지 번호를 1로 초기화하지 않아 실제 결과가 있어도 빈 페이지가 보일 수 있다.

### 6.4 상태 및 예외 처리

| 상태 | 처리 내용 | 판정 |
| --- | --- | --- |
| 정상 | 게시글 목록, 정렬 결과, 페이지 버튼을 표시한다. | 현재 |
| 빈 상태 | 표시할 게시글이 없다는 안내를 표시한다. | 현재 |
| 로딩 | 목록 조회 중 로딩 표시가 없다. | 미구현 |
| API 실패 | 콘솔 오류와 브라우저 경고창을 표시한다. | 현재 |
| 검색·정렬 후 페이지 초과 | 페이지 자동 보정이 없다. | 보완 필요 |

### 6.5 제한 및 반응형

- 서버에서 한 번에 가져오는 게시글은 최대 100개이며, 전체 건수 기반 서버 페이지네이션은 사용하지 않는다.
- 검색어 입력 길이 제한은 없다.
- 모바일에서는 목록 헤더를 숨기고 게시글 행을 작은 화면용 배치로 표시한다.

### 6.6 테스트 조건

| 테스트 ID | 시나리오 | 기대 결과 |
| --- | --- | --- |
| BOARD-T01 | 제목 또는 내용 검색 | 검색어가 포함된 게시글만 표시된다. |
| BOARD-T02 | 조회순 선택 | 조회수가 높은 게시글부터 표시된다. |
| BOARD-T03 | 페이지 버튼 선택 | 선택한 페이지의 게시글 최대 4개가 표시된다. |
| BOARD-T04 | 게시글 행 클릭 | 선택한 게시글 상세 페이지로 이동한다. |
| BOARD-T05 | API 실패 | 경고창이 표시되고 페이지가 강제 종료되지 않는다. |

## 7. PAGE-03 게시글 상세

### 7.1 기본 정보

| 항목 | 내용 |
| --- | --- |
| 페이지 ID | PAGE-03 |
| 페이지명 | 게시글 상세 |
| 접근 URL | `/board/:id` |
| 목적 | 게시글 내용을 확인하고 비밀번호 검증 후 수정 또는 삭제한다. |
| 선행 조건 | URL에 유효한 게시글 ID가 있어야 한다. |
| 데이터/API | `GET /api/posts/{id}`, `POST /api/posts/{id}/verify-password`, `DELETE /api/posts/{id}` |

### 7.2 기능 명세

| 기능 ID | 항목 | 설명 | 상태 |
| --- | --- | --- | --- |
| DETAIL-01 | 게시글 조회 | 제목, 작성일, 조회수, 내용을 표시한다. | 현재 |
| DETAIL-02 | 조회수 증가 | 상세 조회 API 호출 시 조회수가 1 증가한다. | 현재 |
| DETAIL-03 | 목록 이동 | 목록 버튼을 누르면 `/board`로 이동한다. | 현재 |
| DETAIL-04 | 수정 요청 | 비밀번호 모달을 열고 검증 성공 시 `/board/{id}/edit`로 이동한다. | 현재 |
| DETAIL-05 | 삭제 요청 | 비밀번호 검증 성공 시 게시글을 삭제하고 `/board`로 이동한다. | 현재 |
| DETAIL-06 | 모달 초기화 | 수정·삭제 버튼을 누를 때 비밀번호와 오류 메시지를 초기화한다. | 현재 |

### 7.3 입력폼 및 에러 처리

| 상태 | 처리 내용 | 판정 |
| --- | --- | --- |
| 비밀번호 입력 전 | 입력란과 확인·취소 버튼을 표시한다. | 현재 |
| 비밀번호 불일치 | 모달에 불일치 오류를 표시한다. | 현재 |
| 빈 비밀번호 | 별도 사전 검증 없이 API를 호출한다. | 보완 필요 |
| 네트워크·404 오류 | 원인과 관계없이 비밀번호 오류처럼 표시될 수 있다. | 보완 필요 |
| 존재하지 않는 게시글 | 게시글이 존재하지 않는다는 문구를 표시한다. | 현재 |
| 로딩 | 상세 조회 및 비밀번호 검증 중 로딩 표시가 없다. | 미구현 |

### 7.4 제한 및 보안

- 비밀번호 입력 시도 횟수와 요청 빈도 제한이 없다.
- 서버는 게시글 비밀번호를 평문으로 저장하므로 해시 저장 방식으로 개선해야 한다.
- 상세 조회 API가 조회수를 증가시키므로 단순 재조회도 데이터 변경을 일으킨다.

### 7.5 테스트 조건

| 테스트 ID | 시나리오 | 기대 결과 |
| --- | --- | --- |
| DETAIL-T01 | 유효한 ID로 진입 | 내용이 표시되고 조회수가 증가한다. |
| DETAIL-T02 | 잘못된 비밀번호로 수정 | 모달에 오류가 표시되고 이동하지 않는다. |

### 10.3 필터 및 지도 로직

- 카테고리 조건과 구·군 조건은 AND 방식으로 적용한다.
- 카테고리별 개수는 현재 선택된 구·군 조건을 반영한다.
- 필터 변경 시 기존 클러스터를 비우고 새 마커를 생성한다.
- 결과가 0개면 부산 중심, 줌 11로 돌아가고 빈 결과를 표시한다.
- 결과가 1개면 해당 위치를 줌 15로 표시한다.
- 결과가 2개 이상이면 모든 마커가 보이도록 범위를 맞추되 최대 줌은 14로 제한한다.
- 줌 16부터 클러스터를 해제하고 클러스터 반경은 48을 사용한다.
- 클러스터 색상은 개수 30개와 100개를 기준으로 구분한다.
- 팝업에는 카테고리, 구·군, 장소명, 주소와 선택적으로 이미지·전화번호를 표시한다.
- 부산 범위 안의 유효한 위도·경도 데이터만 표시한다.

### 13.1 기본 정보

### 13.3 처리 로직 및 상태

- 프론트 챗봇 요청 타임아웃은 60초이다.
- 백엔드는 기본 Gemini 모델로 `gemini-3.5-flash`를 사용하며 최대 출력 토큰 500, temperature 0.9로 요청한다.
- 일시적인 503 응답은 최대 3회 재시도하고, 최종 외부 모델 오류는 502로 변환한다.
- API 키가 없으면 백엔드가 500 오류를 반환한다.
- 대화 내용은 현재 브라우저 메모리에만 남고 페이지 이동 중에는 유지되지만 새로고침하면 초기화된다.
- 이전 대화 내역을 백엔드에 전달하지 않으므로 각 질문은 독립적으로 처리된다.

## 16. 완료 판정 공통 기준

- 등록된 모든 URL을 직접 입력하거나 메뉴로 이동했을 때 화면이 정상 렌더링되어야 한다.
- 정상 API 응답, 빈 응답, 4xx, 5xx, 타임아웃 각각에서 페이지가 중단되지 않아야 한다.
- 모든 필수 입력은 프론트와 백엔드에서 동일한 기준으로 검증되어야 한다.
- API 요청 중에는 진행 상태를 표시하고 중복 제출을 방지해야 한다.
- 모바일 360px, 태블릿 768px, 데스크톱 1280px 이상에서 핵심 기능을 사용할 수 있어야 한다.
- 키보드만으로 메뉴, 입력폼, 모달, 주요 버튼을 순서대로 조작할 수 있어야 한다.
- 사용자에게 표시되는 오류 문구는 원인과 다음 행동을 이해할 수 있도록 작성해야 한다.
